# Analytical Frameworks Reference

## Market Analysis
- **TAM/SAM/SOM**: Total Addressable Market -> Serviceable Available Market -> Serviceable Obtainable Market. Start with SOM for realistic revenue projections.
- **Porter's Five Forces**: Competitive rivalry, supplier power, buyer power, threat of substitution, threat of new entry. Determines industry attractiveness.
- **Business Model Canvas**: 9 blocks - value proposition, customer segments, channels, customer relationships, revenue streams, key resources, key activities, key partnerships, cost structure.
- **Jobs To Be Done (JTBD)**: People don't buy products, they hire them to do a job. Identify the functional, social, and emotional jobs.
- **Blue Ocean Strategy**: Create uncontested market space rather than competing in existing red oceans. Value innovation = simultaneous pursuit of differentiation AND low cost.
- **Lean Startup**: Build-Measure-Learn loop. Minimum Viable Product -> validated learning -> pivot or persevere.

## Technical Architecture
- **C4 Model**: Context -> Containers -> Components -> Code. Four levels of zoom for system architecture diagrams.
- **SOLID Principles**: Single responsibility, Open/closed, Liskov substitution, Interface segregation, Dependency inversion.
- **12-Factor App**: Codebase, dependencies, config, backing services, build/release/run, processes, port binding, concurrency, disposability, dev/prod parity, logs, admin processes.
- **CAP Theorem**: Consistency, Availability, Partition tolerance - pick two. Guides distributed system design.
- **Architecture Decision Records (ADRs)**: Document context, decision, consequences for each significant architecture choice.
- **Scalability Patterns**: Horizontal scaling, caching (CDN, application, database), message queues, event-driven, CQRS, sharding.

## Risk Assessment
- **Risk Matrix**: Probability (1-5) x Impact (1-5) = Risk Score. Plot on heat map. Focus on high-probability, high-impact quadrant.
- **FMEA (Failure Mode & Effects Analysis)**: For each component - what can fail, how likely, how severe, how detectable? RPN = Severity x Occurrence x Detection.
- **Pre-mortem**: Imagine the project has failed. Work backwards - what went wrong? More effective than traditional risk brainstorming.
- **PESTLE Analysis**: Political, Economic, Social, Technological, Legal, Environmental factors affecting the idea.
- **Dependency Mapping**: Identify all external dependencies (APIs, services, people, regulations). Rate each by criticality and substitutability.
- **Scenario Planning**: Best case, worst case, most likely case. Prepare responses for each.

## Impact Evaluation
- **Theory of Change**: Define long-term impact goal -> work backwards through outcomes -> outputs -> activities -> inputs. Map assumptions at each step.
- **SROI (Social Return on Investment)**: Monetize social outcomes to calculate ratio of social value created per dollar invested.
- **UN Sustainable Development Goals (SDGs)**: 17 global goals. Map project contribution to relevant SDGs for impact framing.
- **Stakeholder Analysis**: Identify all affected parties. Map by interest level and influence. Manage accordingly.
- **Ethics Canvas**: Evaluate group/data subjects affected, potential harms, benefits, transparency, consent, privacy, fairness, accountability.
- **Accessibility Audit (WCAG)**: Perceivable, Operable, Understandable, Robust. Ensure inclusive design.

## Resource Strategy
- **RACI Matrix**: Responsible, Accountable, Consulted, Informed. Clarify roles for each deliverable.
- **Cost-Benefit Analysis**: Quantify all costs and benefits over time. Calculate NPV, ROI, payback period.
- **Make vs Buy**: Build internally vs. use existing solution. Consider total cost of ownership, strategic importance, and available expertise.
- **Critical Path Method**: Identify the longest sequence of dependent tasks. This determines minimum project duration.
- **Bootstrapping Framework**: Revenue-first, minimal external funding. Prioritize: paying customers > grants > angel > VC.
- **Resource Leveling**: Balance resource demand across timeline. Avoid burnout spikes and idle periods.

## Innovation Research
- **Technology Readiness Level (TRL)**: 1 (basic principles) to 9 (proven in production). Assess maturity of key technologies.
- **Gartner Hype Cycle**: Innovation Trigger -> Peak of Inflated Expectations -> Trough of Disillusionment -> Slope of Enlightenment -> Plateau of Productivity.
- **First Mover vs Fast Follower**: First mover advantage (brand, network effects, learning curve) vs. fast follower advantage (learn from mistakes, lower cost, proven market).
- **Platform Thinking**: Build a platform others can build on. Network effects create moats. Two-sided markets.
- **Adjacent Possible**: Innovation happens at the edges of what's currently possible. Look for combinations of existing capabilities.
- **Innovation Diffusion**: Innovators (2.5%) -> Early Adopters (13.5%) -> Early Majority (34%) -> Late Majority (34%) -> Laggards (16%). Cross the chasm between early adopters and early majority.
